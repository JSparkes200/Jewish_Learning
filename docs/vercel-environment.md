# Vercel & GitHub: environment variables and API data

This repo’s **Next.js app** lives in **`web/`**. Vercel must use **Root Directory = `web`**.

## What ships in Git (no Vercel upload)

| Asset | Purpose |
| --- | --- |
| `web/data/corpus-d.ts` | Lexicon `D` for **`POST /api/mcq-choices`**. Generated by `npm run extract:corpus-d` in `web/`; committed so builds are reproducible. **No env var.** |
| `web/data/*.ts` course files | Section drills, stories, comprehension — used by Learn and APIs indirectly. **No env var.** |
| `web/data/auth-reset-email-map.example.json` | **Template only** (fake username/email). Real allowlist must **not** be committed; set as `AUTH_RESET_EMAIL_MAP` on Vercel (see below). |

## Environment variables (set on Vercel)

Open **Vercel → your project → Settings → Environment Variables**. Add each name for **Production** (and **Preview** if you test PRs).

| Variable | Required | Used by |
| --- | --- | --- |
| `NEXT_PUBLIC_SITE_URL` | Recommended | `app/layout.tsx` — `metadataBase`, Open Graph URLs (e.g. `https://your-app.vercel.app`) |
| `AUTH_RESET_EMAIL_MAP` | For password reset | `POST /api/auth/request-reset` — **single-line JSON** object: `{"username":"email@domain.com"}`. Keys are **lowercase** usernames. Copy from `web/data/auth-reset-email-map.example.json`, edit, then **minify to one line** for the dashboard. |
| `RESEND_API_KEY` | Optional | `request-reset` — send real email via [Resend](https://resend.com) |
| `RESEND_FROM` | Optional | Verified sender (default in code: `onboarding@resend.dev`) |
| `KV_REST_API_URL` | Strongly recommended (prod) | `lib/pw-reset-store.ts` — persist reset code hashes across serverless instances |
| `KV_REST_API_TOKEN` | With URL above | Same |
| `DEMO_RESET_CODE` | Never in prod | If exactly `1`, `request-reset` returns `_demoCode` in JSON for debugging only |

**KV:** In Vercel, **Storage → Create Database → KV**, then **Connect** to the project. Vercel usually injects `KV_REST_API_URL` and `KV_REST_API_TOKEN` automatically — you do not paste them from GitHub.

**Secrets:** Do **not** commit `.env.local` or real `AUTH_RESET_EMAIL_MAP` / `RESEND_API_KEY`. `web/.env.example` lists the same names with placeholders.

## API routes summary

| Route | Env vars | Data |
| --- | --- | --- |
| `POST /api/mcq-choices` | None | `web/data/corpus-d.ts` (in repo) |
| `GET` / `PUT` / `DELETE /api/progress` | `KV_*` required for cloud backup | Sanitized Learn progress JSON per Bearer sync key ([cloud-progress.md](./cloud-progress.md)) |
| `POST /api/auth/request-reset` | `AUTH_RESET_EMAIL_MAP`, optional `RESEND_*`, `DEMO_RESET_CODE` | KV optional via `KV_*` |
| `POST /api/auth/confirm-reset` | `KV_*` recommended | KV or in-memory fallback |

## Local development

```bash
cd web
cp .env.example .env.local
# Edit .env.local with real test values (gitignored)
```

## Syncing env to Vercel (CLI, optional)

If you use [Vercel CLI](https://vercel.com/docs/cli):

```bash
cd web
vercel link
# Add secrets interactively:
vercel env add AUTH_RESET_EMAIL_MAP
vercel env add RESEND_API_KEY
# Or pull remote env into .env.local:
vercel env pull .env.local
```

There is **no** committed script that uploads env vars automatically (would require tokens in CI). Use the **dashboard** or **CLI** above.

## GitHub Actions note

CI that runs `npm run build` in `web/` does **not** need secrets for `mcq-choices` or static pages. If you add E2E tests that hit `request-reset`, inject test env vars as **GitHub Actions secrets**, not in the repo.
